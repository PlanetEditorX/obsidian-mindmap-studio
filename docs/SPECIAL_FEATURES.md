# 特殊功能实现说明

## 1. 三种显示模式共享同一份数据

导图、大纲和文章模式不是三个副本。它们都读取 `MindMapDocument.root`：

- 导图模式改变节点层级后，大纲和文章立即反映。
- 大纲模式修改标题后，导图节点立即更新。
- 文章模式编辑正文后，导图和大纲使用同一文本。

模式切换只改变渲染方式，不创建或转换文件。

## 2. 全局显示模式记忆

用户点击导图、大纲或文章模式后，插件保存 `globalDisplayMode`，并通知所有已经打开的导图切换。之后打开父导图、子导图或其他文件，都会使用同一模式。

设置中的“显示哪些模式”只控制按钮是否可见，不会删除数据；至少保留一种模式。

## 3. 只读锁

只读锁按文件保存。锁定后禁止：

- 新增、删除、编辑和拖拽节点。
- 修改尺寸、样式、表格、代码。
- 粘贴和结构重组。
- 撤销、重做写操作。

仍允许：

- 搜索和定位。
- 缩放、平移。
- 打开链接和子导图。
- 查看图片。
- 临时展开折叠。
- 切换三种显示模式。

## 4. 子导图

### 创建

对节点执行“创建子导图”后：

1. 在父导图资源目录下创建独立 `.mindmap` 文件。
2. 父节点写入 `submap.path`。
3. 子文件写入 `navigation.parentPath` 和 `parentNodeId`。
4. 保存父文件后打开子文件。

### 进入

导图模式中，关联子导图的整个节点都是进入区域，用户不需要精确点击文字。拖拽、右键、折叠按钮和内部链接仍保留自身行为。

大纲和文章模式中，节点标题本身是文字链接，不显示重复的“进入子导图”按钮。

### 返回

子导图左上角显示固定悬浮面包屑：

```text
← 父导图 › 当前导图
```

它不随画布缩放或平移，也不改变中心节点外观。返回后通过 `parentNodeId` 定位来源节点。

## 5. 文章编号

只有“具有子节点或关联子导图”的节点才是文章标题。末端节点被视为正文，因此不会出现无意义的编号。

默认编号序列：

```text
第一章
第一节
一、
（一）
1.
（1）
```

标记“文章模式不自动编号”的节点不会显示编号，也不会占用同级序号，适合前言、摘要、注释和附录。

子导图使用父节点深度作为 `articleBaseDepth`，因此进入子文件后不会重新从第一章开始。

## 6. 文章目录

当当前文件是最上层父导图，且存在子导图时，文章模式生成递归目录页。目录读取整个导图族：

- 普通子节点作为同文件目录项。
- 子导图节点作为可跨文件打开的目录项。
- 更深层子导图继续递归。
- 循环引用通过已访问路径集合停止。

点击目录项会打开对应文件、保持文章模式并定位节点。

通读模式在全书目录后连续渲染各级子导图。最顶层导图只作为书名和目录结构来源，不再重复显示为正文；目录按照文章深度缩进，“章”和所属“节”保持清晰的父子层级。每个子导图的正文标题继续使用目录中的章节编号。目录跳转锚点编码完整文件路径，中文名称相近的章节也不会互相跳错。

## 7. 节点文字对齐与尺寸

节点文字支持左、中、右对齐：

- 插件设置提供全局默认值。
- 节点样式可以单独覆盖。
- 导图显示和 SVG 导出使用同一对齐规则。

节点尺寸支持：

- 右下角尺寸手柄拖动。
- 编辑弹窗精确填写宽度和最小高度。
- 双击手柄或右键恢复自动大小。
- 宽度缩小时文字自动换行。
- 高度是最小高度，内容不会被裁切。

## 8. 富文本

文字块支持字符范围级别的：

- 加粗。
- 斜体。
- 下划线。
- 颜色。

编辑过程中，`reconcileRichTextAfterEdit()` 根据公共前缀和后缀迁移旧样式，减少用户插入或删除字符后格式全部丢失的问题。

## 9. 有序混合内容

节点内容不再固定为“文字加一张图片”，而是有序内容块：

```text
文字 → 图片 → 文字 → 图片
```

支持纯图片节点。兼容字段仍会同步，确保旧文件和旧版本导出逻辑可工作。

## 10. 表格与代码

表格支持：

- 手工创建。
- 粘贴 Markdown 表格。
- 把直接子节点转换为表格。
- 列对齐设置。

代码块支持语言名称，并通过 Obsidian Markdown 渲染器显示语法高亮。

## 11. 图片保存与图床

粘贴图片时先保存到当前导图附近的资源目录。用户可以配置多个通用图床：

每个图床使用独立的折叠卡片，默认仅显示名称和启用状态；点击箭头后再展开 API、鉴权、返回字段、自动上传和连通性检测配置，避免多个图床同时占满设置页。

- `POST` 或 `PUT`。
- `multipart/form-data` 或原始二进制。
- 自定义请求头。
- 自定义文件字段名。
- JSON 响应中的 URL 路径。

连接测试上传 1×1 PNG，用于确认请求和 URL 提取配置。

## 12. 自动上传和本地删除安全条件

自动上传在设定延迟后执行。只有满足以下条件才删除本地文件：

1. 所有选中图床均上传成功。
2. 远程地址已写回节点。
3. 当前导图已成功保存。
4. 本地文件仍位于仓库中。
5. 没有其他节点继续引用该本地资源。

任何图床失败都保留本地文件。

## 13. 图片镜像自动容灾

每个图片块可以有多个远程镜像。候选顺序从当前源开始轮转：

```text
当前图床 → 后续镜像 → 前序镜像 → 本地回退
```

加载错误或超时后自动尝试下一地址。成功后：

- 更新 `source`。
- 记录成功时间。
- 保存文档。
- 旧地址继续保留在镜像列表中。

限制：服务器返回 HTTP 200 但内容是错误占位图片时，浏览器仍会认为加载成功，插件无法可靠识别。

## 14. 全局搜索

本地索引覆盖：

- 所有层级子节点和折叠节点。
- 子导图和更深层后代子导图。
- 富文本、备注、标签、链接。
- 表格单元格和代码。
- 图片说明和资源路径。

索引监听文件创建、修改、重命名和删除，并按修改时间增量重建。

`Ctrl/Cmd+Shift+F`：当前父子导图族。

`Ctrl/Cmd+F` 保留给 Obsidian 默认搜索。全库导图搜索通过工具栏的“全局搜索所有导图”按钮打开。

导图族搜索快捷键在正文、标题、节点输入框或其他可编辑区域聚焦时仍然有效，并兼容非英文键盘布局。

点击结果会打开文件、展开祖先、选中节点并居中。

## 15. 主题和彩色分支

内置主题同时设置：

- 背景与图案。
- 根节点和普通节点颜色。
- 字体与字号。
- 连接线样式和颜色。
- 一级分支循环配色。

彩色分支按一级节点分组，同一一级分支的全部后代继承该颜色。

## 16. 连接线从粗到细

渐细模式按当前实际可见最大深度插值：

- 一级连接线使用起始宽度。
- 最深可见连接线达到最小宽度。
- 展开或折叠后重新计算。
- SVG 导出使用相同算法。

每条线同时写入 SVG 属性和内联样式，降低 Obsidian 社区主题覆盖线宽的风险。

## 17. 撤销与重做

可撤销操作统一经过 `mutate()`，包括内容、结构、样式、尺寸、任务状态、表格、代码和子导图引用。历史记录保存文档快照，并限制最大长度，避免长期编辑无限占用内存。

跨文件创建子导图、上传网络图片等外部副作用无法完全通过单文件撤销回滚，因此实现中会先后保存关键状态并提供安全检查。

## 18. Markdown 与 SVG 导出

Markdown 导出保留：

- 层级列表。
- 任务状态。
- 富文本常见样式。
- 图片、表格和代码块。

SVG 导出保留：

- 布局、主题、节点尺寸和对齐。
- 彩色分支。
- 渐细连接线。
- 富文本和任务图标。

## 19. 一键恢复初始设置

设置页的一键恢复只重置插件设置，不删除或修改已有 `.mindmap` 文件。恢复后刷新所有打开视图，并重新使用默认主题、默认显示模式、默认节点样式和默认搜索配置。
