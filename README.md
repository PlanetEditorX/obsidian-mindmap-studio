# MindMap Studio for Obsidian

MindMap Studio 是一个本地优先、可重新打开继续编辑的 Obsidian 思维导图插件。脑图保存为独立的 `.mindmap` JSON 文件，不会在第二次打开时变成 Markdown 源码或静态图片。


## 1.4.0 导图、大纲与文章三种同步模式

同一个 `.mindmap` 文件现在可以使用三种表现方式，三者共用同一棵节点树，不会复制数据：

- **导图模式**：原来的可缩放、可拖动思维导图画布；
- **大纲模式**：按照节点层级显示缩进大纲；
- **文章模式**：根据层级生成目录、章节标题和正文。

在任意模式修改节点，其他模式会立即同步。例如在文章模式把“唐诗”改成“唐代诗歌”，切回导图和大纲后会看到同样的内容。当前使用的模式和只读状态会写入 `.mindmap` 文件，重新打开后保持不变。

### 模式切换与只读锁

工具栏左侧新增模式切换按钮和锁按钮：

```text
导图 ｜ 大纲 ｜ 文章 ｜ 🔓/🔒
```

锁定后只能查看、搜索、打开链接、进入子导图、缩放和平移。新增、删除、拖拽、粘贴、样式修改和节点编辑都会停用。只读状态下仍可临时展开或收起分支，但不会把折叠状态写回文件。

命令面板新增：

```text
切换到导图模式
切换到大纲模式
切换到文章模式
切换导图只读 / 编辑模式
```

### 大纲模式

大纲模式会展开显示所有层级，包括导图中已折叠的节点。可以：

- 直接点击节点文字进行编辑；
- 双击打开完整节点编辑器；
- 添加子节点、删除节点；
- 查看任务状态、备注、子导图入口；
- 点击结果定位到相同节点。

### 文章模式自动编号

文章模式会自动生成目录。编号只应用于**还有下级节点的标题节点**，末端节点作为正文段落，不继续编号。默认层级为：

```text
第一章
  第一节
    一、
      （一）
        1.
          （1）
```

例如：

```text
唐诗（有子级）          → 第一章 唐诗
  李白（有子级）        → 第一节 李白
    诗歌风格（有子级）  → 一、诗歌风格
      浪漫主义（末端）  → 正文，不再编号
```

同级末端节点不会占用编号序列。节点编辑器和右键菜单中新增“文章模式不自动编号”，适合前言、摘要、注释、附录等。被跳过的节点不会占用后续章、节编号。

### 显示模式设置与一键还原

在插件设置的“显示模式”区域，可以隐藏不需要的模式。例如仅启用“导图”和“文章”，工具栏中将不再显示“大纲”。至少会保留一种模式，并且可以设置新建脑图默认进入哪种模式。

设置页底部新增“一键还原所有插件设置”。它会恢复主题、显示模式、资源目录、图床、搜索和编辑选项，但不会删除或修改任何 `.mindmap` 文件。

## 1.3.1 父导图与子导图递归搜索修复

搜索现在会识别真实的父子导图关系，而不是只把每个 `.mindmap` 文件当作互不相关的文件。

例如：

```text
古诗.mindmap
└── 唐诗（子导图）
    └── 李白
```

在 `古诗.mindmap` 中按 `Ctrl/Cmd + F` 搜索“李白”，插件会递归读取“唐诗”子导图并返回对应节点。点击结果后会打开 `唐诗.mindmap`、展开相关分支并定位到“李白”。结果层级会显示为：

```text
古诗 › 唐诗 › 李白
```

已有子导图**不需要重新创建**。打开当前导图搜索时，插件会即时刷新当前父导图及其全部后代子导图；只要父节点仍保存 `submap.path`，或者子导图仍保存 `navigation.parentPath`，旧文件就可以自动识别。即使从“唐诗”子导图发起搜索，也会先向上找到“古诗”，再搜索完整导图树。

两个搜索入口分别是：

```text
Ctrl/Cmd + F          当前导图及递归关联的全部子导图
Ctrl/Cmd + Shift + F  整个仓库中的全部思维导图
```

当前导图族搜索会按需更新相关文件，不要求先手动重建整个仓库索引。全局搜索仍使用本地增量索引库，并支持用多个层级关键词搜索，例如“古诗 唐诗 李白”。

## 1.3.0 全局搜索与本地索引库

新增跨整个 Obsidian 仓库的思维导图搜索。点击导图工具栏中的“全局搜索”按钮，或使用：

```text
Ctrl/Cmd + Shift + F
```

搜索范围包括所有 `.mindmap` 文件中的：

- 根节点、普通子节点和任意深度的后代节点；
- 已折叠但当前没有显示的分支；
- 每个独立子导图文件中的节点；
- 多段富文本、备注、标签和链接；
- 表格表头及单元格；
- 代码语言和代码内容；
- 子导图名称及路径；
- 图片替代文字和资源路径。

点击搜索结果后，插件会打开对应 `.mindmap` 文件，自动展开目标节点的祖先分支，并将目标节点居中选中。搜索结果会显示导图名称、文件路径、节点层级路径和命中的内容类型。

### 是否依赖数据库

插件不使用外部服务器或云数据库，而是在插件目录中维护一个本地增量索引文件：

```text
.obsidian/plugins/mindmap-studio/mindmap-search-index.json
```

它相当于只服务于思维导图搜索的本地索引库。启动时只读取发生变化的文件；之后监听 `.mindmap` 文件的新建、修改、重命名和删除并增量更新。正常搜索不会每次重新读取所有导图。

索引只保存可搜索文字、节点 ID、文件路径和层级信息，不保存图片二进制，也不会上传网络。可以在插件设置的“全局搜索索引”区域查看收录数量、调整结果上限或手动重建索引。

## 1.2.1 连线渐细修复

“从粗到细”现在按照当前导图实际可见的最深层级计算。即使导图只有两层或三层，一级分支仍使用“起始粗细”，最深一层会达到“末端最细宽度”，不会再因为固定按五层计算而看不出变化。

如果节点分支被折叠，渐细计算会以当前可见层级为准；展开更深分支后会自动重新计算。


## 为什么不再叫 MindMap Canvas

旧名称中的 Canvas 只是指可缩放、可平移的编辑区域，但容易与 Obsidian 自带 Canvas 功能混淆。从 0.9.0 起，插件正式更名为 **MindMap Studio**：

- 插件显示名称：`MindMap Studio`
- 插件 ID 和安装目录：`mindmap-studio`
- 脑图文件后缀：`.mindmap`（保持不变）
- 旧插件设置：首次启动新版时会尝试从 `mindmap-canvas/data.json` 自动迁移

升级后请关闭并移除旧的 `mindmap-canvas` 插件目录，避免两个插件同时注册 `.mindmap` 文件类型。

## 1.2.0 重点功能

### 多套主题模板

“当前脑图外观”窗口顶部新增主题选择区，内置 10 套独立设计的脑图主题：

- 经典靛蓝
- 深海蓝
- 森林绿
- 日落橙
- 薰衣草
- 糖果缤纷
- 纸张笔记
- 极简墨色
- 暗夜霓虹
- 薄荷清新

主题不是只替换背景，而是会一起应用：

- 画布背景和网格/点阵；
- 中心主题颜色和文字颜色；
- 普通节点背景、文字和边框；
- 一级分支循环配色；
- 字体和字号；
- 连线颜色、类型和粗细模式。

选择主题后，仍可以继续修改任意单项。当前脑图的主题和后续微调都会写入 `.mindmap` 文件，重新打开后保持不变。插件设置页也提供“默认主题”，用于之后新建的脑图。

### 连线从粗到细

连线新增两种粗细模式：

```text
统一粗细
从粗到细
```

“从粗到细”模式使用两个参数：

- **起始粗细**：中心主题到一级分支的线宽；
- **末端最细宽度**：深层节点连线的最小宽度。

插件会根据节点层级平滑递减线宽。例如设置起始 4.5 px、末端 1 px 后，一级分支最粗，二级、三级逐渐变细，深层分支保持在 1 px，不会无限变细。

彩色分支开启后，同一个一级分支下面的所有后代会沿用相同的连线和节点边框颜色。SVG 导出和 Markdown 静态预览也会保留主题及渐细连线。

旧用户升级后，原有全局配置默认继续使用统一粗细和彩色分支关闭，不会自动改变现有脑图外观。新建安装默认使用经典靛蓝主题和渐细连线。

## 1.1.0 重点功能

### 图床镜像自动故障转移

同一张图片上传到多个图床后，插件会把所有网址按优先级保存在图片块中。打开可编辑脑图并加载图片时：

```text
当前主地址加载失败或超过等待时间
→ 尝试下一个图床地址
→ 成功后立即显示备用图片
→ 将成功地址写成新的主地址
→ 自动保存 .mindmap 文件
```

例如镜像顺序为图床 A、图床 B、图床 C：

- A 失效时自动切换到 B；
- B 以后失效时继续尝试 C；
- 如果开启本地回退且本地副本仍存在，远程镜像全部失败后会使用本地图片；
- 原镜像地址不会从列表删除，便于恢复后手动切换或继续作为备用地址。

设置位置：

```text
Obsidian 设置 → 第三方插件 → MindMap Studio → 图片与图床
```

可设置：

- 是否启用远程图片自动故障转移；
- 单个镜像等待时间，范围 2–30 秒；
- 是否将本地副本作为最后回退。

故障转移在图片实际加载时触发。图床如果返回 HTTP 200 的占位图或错误图片，而不是加载错误，插件无法仅凭浏览器图片事件判断内容是否正确。

## 1.0.0 重点功能

### 可排序的文字与图片内容块

节点不再固定为“一个标题加一张图片”，而是由多个内容块组成。内容块可以上下移动：

- 纯文字
- 纯图片
- 图片 → 文字
- 文字 → 图片
- 文字 → 图片 → 文字
- 多张图片和多段文字任意穿插

编辑节点时，可以删除默认的“新节点”文字块。只要还保留一个有效图片块，节点就会作为纯图片节点保存。

每个文字块都支持局部富文本：选择任意字符后设置加粗、斜体、下划线或颜色，下方实时预览。不同文字块可以分别设置格式。

### 图片点击放大

在可编辑脑图中点击任意图片会打开大图预览：

- 鼠标滚轮缩放
- `+` / `−` 调整缩放
- `100%` 恢复原始比例
- 双击图片恢复 100%
- 大图可在预览窗口中滚动查看

### 本地图片资源

粘贴图片或点击“保存到仓库”时，图片默认保存到：

```text
当前脑图所在目录 / 设置的资源文件夹 /
```

例如：

```text
Projects/
├── Plan.mindmap
└── MindMap Assets/
    └── Plan-20260720-123456.png
```

资源文件夹可在插件设置中修改，默认是 `MindMap Assets`。

### 多图床、自动上传与安全删除

插件可以配置多个通用 HTTP 图床，每个图床分别设置：

- 名称和启用状态
- 上传 API 地址
- `POST` 或 `PUT`
- `multipart/form-data` 或原始二进制
- multipart 文件字段名
- 自定义请求头 JSON
- 返回图片地址字段，例如 `data.url`、`result.image`

每个图床都有“检测 API 连通性”按钮。检测会实际上传一张 1×1 透明 PNG，以验证鉴权、请求格式和返回网址解析，因此图床中可能留下一个很小的测试文件。

手动上传时会弹出图床选择窗口，可以选择一个或多个目标。第一项成功返回的网址作为节点当前显示地址，其余网址作为镜像保存在 `.mindmap` 文件中。

自动上传采用本地优先流程：

```text
粘贴图片
→ 先写入当前脑图目录的资源文件夹
→ 等待设定的延迟时间
→ 同时上传到选中的一个或多个图床
→ 全部成功后把节点引用替换为第一项远程网址
→ 先保存脑图
→ 确认本地图片未被其他脑图引用后再删除本地文件
```

只要任一目标失败，节点继续使用本地图片，本地文件不会删除。已经成功的图床网址仍会作为镜像记录，方便后续排查或重试。

设置位置：

```text
Obsidian 设置 → 第三方插件 → MindMap Studio → 图片与图床
```

可设置：

- 粘贴图片后自动上传开关
- 0–300 秒上传延迟
- 默认自动上传目标，可多选
- 全部成功后是否删除本地图片
- 多个图床配置

自定义请求头会保存在插件的 `data.json` 中。不要在共享仓库或不可信设备中保存敏感令牌。

## 安装

1. 下载并解压安装包。
2. 将 `mindmap-studio` 文件夹放入：

```text
你的仓库/.obsidian/plugins/
```

最终目录：

```text
.obsidian/plugins/mindmap-studio/
├── main.js
├── manifest.json
└── styles.css
```

3. 删除或停用旧目录：

```text
.obsidian/plugins/mindmap-canvas/
```

4. 重启 Obsidian。
5. 在“设置 → 第三方插件”中启用 **MindMap Studio**。

## 节点内容编辑

双击节点或按 `F2` 打开编辑窗口。

### 添加内容块

点击：

- `+ 文字`
- `+ 图片`

每个内容块右上角有：

- 上移
- 下移
- 删除

图片块支持：

- 输入仓库路径
- 输入 `[[图片文件]]`
- 输入网络图片 URL
- 从电脑选择并保存到仓库
- 从电脑选择并上传到一个或多个图床
- 将当前本地图片手动上传到一个或多个图床
- 查看已经保存的远程镜像地址

### 创建纯图片节点

1. 新建节点。
2. 添加图片块并选择图片。
3. 删除默认文字块，或者清空文字后删除该文字块。
4. 保存并关闭。

节点将只显示图片，不会强制保留“新节点”。

## 子导图目录

假设父导图位于：

```text
Projects/Plan.mindmap
```

在节点“市场分析”上创建子导图后，默认生成：

```text
Projects/
├── Plan.mindmap
└── MindMap Assets/
    └── Plan/
        └── 市场分析.mindmap
```

子导图顶部会显示返回父导图的导航按钮，并保存父文件路径和来源节点 ID。

## 数据兼容

1.0.0 文件数据版本为 `8`。

旧版数据会自动转换为内容块：

- 旧 `image` 字段转换成图片块
- 新图片块可保存 `localSource` 和多个 `remoteSources`
- 旧 `text` / `richText` 字段转换成文字块
- 同时有图片和文字时，默认转换为“图片 → 文字”

旧字段仍会同步保留，用于兼容较早版本和部分导出逻辑。
