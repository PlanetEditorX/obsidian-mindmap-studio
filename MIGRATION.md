# 升级到 MindMap Studio 1.4.0

1. 关闭 Obsidian 中的 MindMap Studio。
2. 用 1.4.0 安装包中的 `mindmap-studio` 文件夹覆盖旧目录。
3. 重启 Obsidian 或重新启用插件。

现有 `.mindmap` 文件可以直接打开。数据版本会在下次保存时从版本 9 升级为版本 10，新增的只是显示模式、只读状态和文章编号跳过标记，不会改变原来的节点层级或内容。

升级后默认显示导图、大纲和文章三种模式。可以进入：

```text
Obsidian 设置 → 第三方插件 → MindMap Studio → 显示模式
```

隐藏不需要的模式，并指定新建脑图的默认模式。旧文件首次打开仍默认使用导图模式。

文章模式的编号只应用于有子级的节点。需要作为前言、注释或附录的节点，可在节点编辑器中开启“文章模式不自动编号”。

设置页的“一键还原所有插件设置”会清除图床等插件配置，请在使用前确认；它不会删除任何脑图文件或图片资源。

# 升级到 MindMap Studio 1.3.1

1. 关闭 Obsidian 中的 MindMap Studio。
2. 用 1.3.1 安装包中的 `mindmap-studio` 文件夹覆盖旧目录。
3. 重启 Obsidian 或重新启用插件。

`.mindmap` 文件格式没有变化，已有子导图无需重新创建。

搜索快捷键现在是：

```text
Ctrl/Cmd + F          当前导图及递归子导图
Ctrl/Cmd + Shift + F  整个仓库的全局搜索
```

打开 `Ctrl/Cmd + F` 搜索时，插件会即时读取当前父导图及其关联子导图，因此“古诗 → 唐诗 → 李白”这类旧结构不需要先手动重建索引。全局索引格式会自动升级为版本 2，用于显示完整的父子导图层级。

# 升级到 MindMap Studio 1.3.0

1. 关闭 Obsidian 中的 MindMap Studio。
2. 用 1.3.0 安装包中的 `mindmap-studio` 文件夹覆盖旧目录。
3. 重启 Obsidian 或重新启用插件。

`.mindmap` 文件的数据结构版本仍为 9，现有脑图不需要转换。首次启动 1.3.0 后，插件会在后台建立本地搜索索引。导图数量较多时，最初几秒内搜索结果可能逐步增加，但编辑功能不受影响。

## 新增本地索引文件

插件目录会生成：

```text
.obsidian/plugins/mindmap-studio/mindmap-search-index.json
```

该文件是可重新生成的缓存，不是脑图源数据：

- 删除它不会删除任何脑图；
- 插件下次启动会自动重建；
- 不需要纳入版本控制；
- 不会上传到图床或其他网络服务。

## 搜索入口

- 导图工具栏中的全局搜索按钮；
- Obsidian 左侧功能区的搜索按钮；
- 命令面板中的“全局搜索所有思维导图”；
- 快捷键 `Ctrl/Cmd + Shift + F`。

旧的 `Ctrl/Cmd + F` 继续只搜索当前打开的导图。
